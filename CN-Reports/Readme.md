# CN Labs — Computer Networks

This folder contains all Computer Networks lab materials, separated into Software (Packet Tracer) labs and Hardware labs.

## Purpose

- Document foundational and advanced computer networks labs in a structured and professional way.  
- Showcase learning progression from network simulation to real hardware implementation.  
- Maintain lab work performed in controlled and isolated environments.

## Important Note on Contents

- **Software Labs (PART A — Packet Tracer)**  
  Original step‑by‑step PDF lab files are stored in `CLG-Labs/` and corresponding polished, text‑based reports are in `CLG-Lab-Reports/`.

- **Hardware Labs (PART B — Physical Equipment)**  
  Only polished, text‑based reports are provided in `CLG-Lab-Reports/`. Hardware lab step‑by‑step PDFs are **not** included here (reporting is focused on methodology, configurations, and results).

## Contents

### Current CN Lab Topics

**PART A — SOFTWARE LABS (Packet Tracer)**  
- `01-Network-Basic-Ping`            — Construct network with hub, switches, routers; test ping  
- `02-Router-Interconnect`           — Connect two networks via router; ping within & between networks  
- `03-ARP-ICMP-Sniffing`             — Observe ARP protocol and ICMP packets using sniffers  
- `04-DHCP-Network`                  — DHCP server setup and a network with 5 routers

**PART B — HARDWARE LABS (Physical Lab)**  
- `05-Telnet-Basic-Connectivity`     — Telnet and basic connectivity in switch mode  
- `06-VLAN-Isolation`                — VLAN isolation: intra-VLAN allowed, inter-VLAN blocked  
- `07-ACL-Demonstration`             — Demonstrate Access Control Lists on router  
- `08-Inter-VLAN-Communication`      — Inter-VLAN routing and communication  
- `09-Router-On-A-Stick`             — Router-on-a-Stick configuration  
- `10-OSPF-Demonstration`            — Demonstrate OSPF routing protocol

---

> All work is performed in controlled virtual or lab environments. Software lab PDFs include step-by-step instructions and outputs; polished reports are text-only and emphasize objective, methodology, findings, analysis, and lessons learned.
